# Exercise Pattern Library

## Purpose

An `ExercisePattern` defines the teaching structure of one step within a lesson session.

It is not a content template. It defines: what skill is practised, what input the student receives, what output is expected, how it is scored, and how long it takes.

Content (the actual text, audio, questions, prompts) is generated by AI or deterministically for each individual exercise instance.

SpeakPath treats a lesson as a classroom sequence:

```
Teach -> Practice -> Feedback
```

The skill focus answers what the student is learning, such as listening,
reading, speaking, writing, vocabulary, or grammar. The exercise pattern answers
how the student practises it, such as matching, gap fill, email reply, workplace
chat, or spoken response. Do not use `ActivityType` as the product experience.
It is an implementation routing detail.

The same active exercise patterns can be used by Today's Lesson and Practice
Gym. Today's Lesson should prefer ready buffered `LearningSession` exercises.
Practice Gym should prefer ready `PracticeActivityCache` rows before any
on-demand generation.

## ExercisePattern Model

```json
{
  "key": "listen_and_gap_fill",
  "name": "Listen and Gap Fill",
  "primarySkill": "Listening",
  "secondarySkills": ["Vocabulary", "Grammar"],
  "inputMode": "Audio",
  "outputMode": "Text",
  "interactionMode": "Individual",
  "estimatedMinutes": 4,
  "difficulty": "B1",
  "teachingPurpose": "notice useful workplace phrases from audio",
  "workplaceContext": true,
  "scoringDimensions": ["listening comprehension", "vocabulary accuracy"]
}
```

### Fields

| Field | Type | Notes |
|---|---|---|
| key | string | Unique identifier used in code and DB |
| name | string | Human-readable display name |
| primarySkill | string | Main skill being practised |
| secondarySkills | string[] | Additional skills involved |
| inputMode | InputMode | Text, Audio, Image, None |
| outputMode | OutputMode | Text, Audio, Selection, None |
| interactionMode | string | Individual, Simulation |
| estimatedMinutes | int | Default time estimate |
| teachingPurpose | string | Why this exercise is used in a lesson |
| workplaceContext | bool | Whether it uses a workplace scenario |
| scoringDimensions | string[] | What is evaluated |

## Pattern Library

### INPUT / COMPREHENSION

#### `listen_and_answer`
- **Primary:** Listening
- **Secondary:** Vocabulary
- **Input:** Audio
- **Output:** Text (short answers)
- **Purpose:** check understanding of a workplace audio message
- **Minutes:** 4

#### `listen_and_gap_fill`
- **Primary:** Listening
- **Secondary:** Vocabulary, Grammar
- **Input:** Audio
- **Output:** Text (fill in blanks)
- **Purpose:** notice useful phrases from audio; train active listening
- **Minutes:** 4

#### `listen_and_take_notes`
- **Primary:** Listening
- **Secondary:** Writing, Summarising
- **Input:** Audio
- **Output:** Text (notes)
- **Purpose:** train active note-taking from workplace messages
- **Minutes:** 5

#### `read_and_answer`
- **Primary:** Reading
- **Secondary:** Vocabulary
- **Input:** Text (workplace message, email, report excerpt)
- **Output:** Text (short answers)
- **Purpose:** comprehension of written workplace communication
- **Minutes:** 4

#### `read_and_summarise`
- **Primary:** Reading
- **Secondary:** Writing, Summarising
- **Input:** Text
- **Output:** Text (summary)
- **Purpose:** extract key information and express it concisely
- **Minutes:** 5

#### `read_teams_thread_and_identify_tone`
- **Primary:** Reading
- **Secondary:** Vocabulary, Workplace Tone
- **Input:** Text (Teams-style chat thread)
- **Output:** Selection + Text (identify tone, explain choice)
- **Purpose:** develop awareness of professional digital communication tone
- **Minutes:** 4

### VOCABULARY / COLLOCATION

#### `phrase_match`
- **Primary:** Vocabulary
- **Secondary:** None
- **Input:** Text (phrase list + meanings)
- **Output:** Selection (match pairs)
- **Purpose:** introduce or review key workplace phrases before deeper use
- **Classroom structure:** word or phrase cards with meaning and context first; matching practice second; feedback after submit
- **Minutes:** 2

#### `collocation_match`
- **Primary:** Vocabulary
- **Secondary:** Grammar
- **Input:** Text (word pairs to match)
- **Output:** Selection
- **Purpose:** build natural English word combinations (make a decision, raise a concern)
- **Minutes:** 3

#### `sentence_completion`
- **Primary:** Vocabulary
- **Secondary:** Grammar
- **Input:** Text (incomplete sentences)
- **Output:** Text
- **Purpose:** practise using target vocabulary in context
- **Minutes:** 3

#### `choose_best_phrase`
- **Primary:** Vocabulary
- **Secondary:** Workplace Tone
- **Input:** Text (scenario + 3–4 phrase options)
- **Output:** Selection
- **Purpose:** choose the most appropriate workplace phrasing
- **Minutes:** 3

#### `gap_fill_with_workplace_phrase`
- **Primary:** Vocabulary
- **Secondary:** Grammar
- **Input:** Text (paragraph with gaps, word bank provided)
- **Output:** Text
- **Purpose:** practise target phrases in a realistic workplace context
- **Classroom structure:** one focused vocabulary or grammar explanation first; fill-in-the-blank practice second; feedback after submit
- **Minutes:** 4

#### `direct_to_professional_phrase`
- **Primary:** Vocabulary
- **Secondary:** Workplace Tone
- **Input:** Text (casual/direct phrase to rewrite)
- **Output:** Text
- **Purpose:** convert informal or blunt phrasing to professional English
- **Minutes:** 3

### GRAMMAR / LANGUAGE FOCUS

#### `error_correction`
- **Primary:** Grammar
- **Secondary:** Vocabulary
- **Input:** Text (sentences with errors)
- **Output:** Text (corrected sentences)
- **Purpose:** notice and fix common grammar mistakes
- **Minutes:** 4

#### `sentence_reordering`
- **Primary:** Grammar
- **Secondary:** Vocabulary
- **Input:** Text (scrambled words or clauses)
- **Output:** Text (correct order)
- **Purpose:** practise sentence structure and word order
- **Minutes:** 3

#### `guided_sentence_building`
- **Primary:** Grammar
- **Secondary:** Vocabulary
- **Input:** Text (prompts: topic + required phrase)
- **Output:** Text (student composes sentence)
- **Purpose:** guided production of grammatically correct workplace sentences
- **Minutes:** 4

#### `rewrite_for_tense`
- **Primary:** Grammar
- **Input:** Text (present-tense paragraph)
- **Output:** Text (rewritten in target tense)
- **Purpose:** practise tense accuracy in workplace narratives
- **Minutes:** 4

#### `rewrite_for_polite_tone`
- **Primary:** Grammar
- **Secondary:** Workplace Tone
- **Input:** Text (direct or blunt message)
- **Output:** Text (polite rewrite)
- **Purpose:** practise formal request structures, modal verbs, softening language
- **Minutes:** 4

### WORKPLACE COMMUNICATION

#### `teams_chat_simulation`
- **Primary:** Writing
- **Secondary:** Workplace Tone, Vocabulary
- **Input:** Text (Teams-style message from colleague/manager)
- **Output:** Text (student reply)
- **Purpose:** practise concise, professional digital communication
- **Minutes:** 5
- **Evaluation rubric (`activity_evaluate_teams_chat_simulation`):** in addition to grammar/vocabulary/tone, the AI evaluates whether the reply addresses the exercise's `learningGoal` (e.g. asking for clarification, giving a status update), whether the tone is appropriate (not over-apologising, not too casual/formal), whether the message is clear, and — if relevant to the goal — whether the student asked a useful clarifying question. The `learningGoal` is shown to the student as a small "Goal" label above the chat thread.
- **See:** [TeamsChatSimulation design](#teams-chat-simulation)

#### `email_reply`
- **Primary:** Writing
- **Secondary:** Grammar, Vocabulary, Tone
- **Input:** Text (workplace email)
- **Output:** Structured email reply — separate `subject` and `body` fields (`EmailReplyComponent`)
- **Purpose:** structured workplace writing with correct format and tone
- **Minutes:** 7
- **Submission shape:** `{ "subject": "...", "body": "..." }`, evaluated by `activity_evaluate_email_reply` (subject clarity/relevance + body grammar/vocabulary/tone/structure)

#### `meeting_followup`
- **Primary:** Writing
- **Secondary:** Summarising
- **Input:** Text (brief meeting notes or scenario description)
- **Output:** Text (follow-up message or action list)
- **Purpose:** professional summary and follow-up communication
- **Minutes:** 6

#### `incident_update`
- **Primary:** Writing
- **Secondary:** Grammar, Vocabulary
- **Input:** Text (situation description)
- **Output:** Text (professional incident update message)
- **Purpose:** concise professional reporting under pressure
- **Minutes:** 5

#### `clarification_request`
- **Primary:** Writing
- **Secondary:** Vocabulary, Tone
- **Input:** Text (ambiguous instruction or message)
- **Output:** Text (polite clarification request)
- **Purpose:** practise asking for clarification without appearing incompetent
- **Minutes:** 5

#### `escalation_message`
- **Primary:** Writing
- **Secondary:** Tone, Vocabulary
- **Input:** Text (unresolved issue scenario)
- **Output:** Text (escalation message to manager)
- **Purpose:** professional escalation language and tone calibration
- **Minutes:** 6

- **Minutes:** 5

### MICRO LESSON / INPUT TEACHING

#### `micro_lesson_phrases`
- **Primary:** Vocabulary
- **Secondary:** Grammar
- **Input:** Text (AI teacher explanation: target phrases, usage rules, examples)
- **Output:** None (read-only / listen-only step)
- **Purpose:** teach before practise — introduce phrases the student will use in subsequent exercises
- **Minutes:** 2

#### `micro_lesson_mistake`
- **Primary:** Reflection
- **Secondary:** Grammar
- **Input:** Text (AI explanation of a common mistake pulled from student memory)
- **Output:** None (read-only)
- **Purpose:** short targeted correction of a recurring error before the student practises again
- **Minutes:** 2

#### `micro_lesson_dialogue`
- **Primary:** Listening
- **Secondary:** Vocabulary
- **Input:** Text or Audio (short example workplace dialogue)
- **Output:** None (read-only / listen-only)
- **Purpose:** model natural workplace English before the student attempts a similar task
- **Minutes:** 2

### CALL MODE / OPEN SPEAKING (future P2)

These patterns are designed now but not implemented until the Call Mode sprint.

#### `call_mode_single_turn`
- **Primary:** Speaking
- **Secondary:** Listening, Vocabulary
- **Input:** Audio (AI speaks first — question or instruction)
- **Output:** Audio (student replies)
- **Purpose:** realistic phone/meeting turn: student must respond to an AI-generated spoken prompt
- **Interaction:** Simulation (AI-first)
- **Minutes:** 3

#### `call_mode_multi_turn`
- **Primary:** Speaking
- **Secondary:** Listening, Vocabulary, Tone
- **Input:** Audio (AI-driven multi-turn conversation)
- **Output:** Audio (multiple student turns)
- **Purpose:** full workplace AI call simulation: manager update request, complaint, or meeting follow-up; AI leads; student responds across 3–5 turns; post-call transcript and feedback
- **Interaction:** Simulation (AI-first, multi-turn)
- **Minutes:** 8

Workplace call scenarios:
- Manager asks for project update
- Customer complaint call
- Interview practice (mock interview)
- Meeting follow-up confirmation
- Explaining a delay by phone

Post-call evaluation dimensions:
- Clarity and structure of responses
- Vocabulary range and appropriateness
- Professional tone throughout the call
- Response completeness (addressed all questions)
- Grammar accuracy in spoken output

## Teams Chat Simulation

`teams_chat_simulation` is a high-priority pattern for workplace English.

### Content JSON Shape

```json
{
  "patternKey": "teams_chat_simulation",
  "scenario": "Your manager asks about the status of a document submission.",
  "chatThread": [
    {
      "sender": "Manager",
      "message": "Hi, can you give me an update on the IFC submittal? We were expecting it yesterday.",
      "timestamp": "09:14"
    }
  ],
  "studentRole": "Document Controller",
  "targetPhrases": ["I apologise for the delay", "I'll have it ready by", "Could you please confirm"],
  "toneGuidance": "Professional and solution-focused. Avoid blaming others.",
  "wordLimit": 60,
  "difficulty": "B1"
}
```

### Expected UI

- Teams-style chat bubble layout (sender above, student reply below)
- Character/word counter visible while typing
- Target phrases shown as hints (collapsible)
- Submit → AI evaluates reply
- Feedback: tone score, phrase usage, conciseness rating, improved version

### Evaluation Dimensions

- Professionalism of tone
- Use of target phrases
- Conciseness (within word limit)
- Response completeness (answers the manager's question)
- Grammar accuracy

### How it differs from WritingScenario / email_reply

- Short, conversational format (not a formal email)
- Turn-based context (a real message to reply to)
- Word limit enforced (60–100 words)
- Tone specifically evaluated for digital workplace chat norms
- Can be embedded inside a lesson session or used in Practice Gym

## Frontend Renderer Contract

Pattern-generated activities are rendered by `ActivityDto.interactionMode`, not by `ActivityType`.

`ActivityLessonComponent` passes the full activity DTO into `ExerciseRendererComponent`. The renderer shell parses the pattern `contentJson`, normalises minor schema differences between legacy DTO fields and pattern payloads, and dispatches to one small interaction component:

| InteractionMode | Renderer | MVP pattern support |
|---|---|---|
| `ReadOnly` | `ReadOnlyStepComponent` | `lesson_reflection` |
| `FreeTextEntry` | `FreeTextEntryComponent` | `spoken_response_from_prompt`, free-text fallback |
| `MatchingPairs` | `MatchingPairsComponent` | `phrase_match` |
| `GapFill` | `GapFillComponent` | `gap_fill_workplace_phrase` |
| `AudioAndFreeText` | `AudioAndFreeTextComponent` | `listen_and_answer` |
| `AudioAndGapFill` | `AudioAndGapFillComponent` | `listen_and_gap_fill` |
| `ChatReply` | `ChatReplyComponent` | `teams_chat_simulation` |
| `EmailReply` | `EmailReplyComponent` | `email_reply` |
| `FreeTextEntry` | `FreeTextEntryComponent` | `open_writing_task` |
| `AudioResponse` | speaking recording UI (`speakingScenario`/`speakingRecord` blocks, activity-type-keyed, not via `ExerciseRendererComponent`) | `speaking_roleplay_turn` |

Each renderer emits one structured answer payload to the parent. The parent submits via the existing activity attempt endpoints. The backend routes by `MarkingMode` through `IPatternEvaluationRouter`. The `ActivityFeedbackDto` returns `patternEvaluation: PatternEvaluationDto | null` — non-null for pattern-keyed activities. The frontend renders `PatternEvaluationResultComponent` when present; legacy feedback sections gate on `!patternEvaluation`.

`contentJson` is returned to the frontend only for pattern-keyed activities. Legacy listening activities do not expose raw content JSON before submission because it can contain transcripts or expected answers.

## Usage in Session Generator

When the session generator selects exercise patterns, it applies these rules:

1. Sessions begin with a low-anxiety exercise (phrase_match, read_and_answer, listen_and_answer) or a micro lesson.
2. Micro lesson patterns (`micro_lesson_*`) always precede the first practice exercise of the same type.
3. Productive tasks (email_reply, teams_chat_simulation, spoken_response_from_prompt) always follow input and vocabulary exercises.
4. Reflection/review patterns always appear last in the session.
5. The same pattern key must not appear twice in one session.
6. Vocabulary/review patterns may appear early if the student has items due in their vocabulary queue.
7. Pattern difficulty must match student CEFR level (±0.5 levels allowed).
8. `call_mode_*` patterns are only valid in Practice Gym Call Mode or a dedicated Call Mode session type — they do not appear in standard guided lesson sessions.
