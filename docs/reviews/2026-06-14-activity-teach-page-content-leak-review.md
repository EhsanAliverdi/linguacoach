## Finding: root cause

The Step 1-5 migration (already landed) correctly split rendering into
`ActivityTeachPageComponent` / `ActivityPracticePageComponent` /
`ActivityFeedbackPageComponent`, and all 10 activity types now go through the
pattern engine via `PatternBackedPresenter`.

But `PatternBackedPresenter.teachContent()`
(`pattern-backed.presenter.ts:15-40`) pulls the **entire** `contentJson` blob
into the `patternLearning` teach view model, including fields that are
exercise material, not teaching material:

- `targetPhrases` — phrases the student should *use while doing the exercise*
- `targetVocabulary` — vocab the exercise tests
- `exampleText` / `exampleReply` — a worked example of the exercise answer
- `commonMistakeToAvoid` — feedback-adjacent guidance tied to the exercise task
- `scenario` / `situation` / `audience` / `tone` / `speakerRole` /
  `listenerRole` — these describe the **exercise scenario** (the task setup),
  not a grammar/communication lesson

`activity-teach-page.component.html` (`patternLearning` case, lines 228-341)
renders **all** of these on Page 1. The CTA button is literally "Start
practice" — i.e. Page 1 today is "here's the exercise setup and hints, click
to start the exercise", which is exactly the practice page's content shown
early, not a teaching page.

Meanwhile `practiceContent()` (`pattern-backed.presenter.ts:42-47`) returns
almost nothing (`{ block: 'exerciseRenderer', skillBadge }`) — Page 2 renders
only `app-exercise-renderer` + title + `instructionInSourceLanguage`
(`activity-practice-page.component.html:63-95`). So the scenario/phrases/
vocab/example that *should* support Page 2 are stuck on Page 1 instead.

**There is currently no "Learn" content at all** — no grammar explanation, no
"how to politely ask for X" micro-lesson. `contentJson` (as generated by AI
prompts today) only contains exercise-setup fields. The user's "Learn" page 1
concept does not exist yet in the data model or AI prompts.

## Two distinct problems

1. **Misplacement**: exercise-setup fields (`scenario`, `situation`,
   `targetPhrases`, `targetVocabulary`, `exampleText`,
   `commonMistakeToAvoid`, `speakerRole`/`listenerRole`, `audience`/`tone`)
   are rendered on Page 1 but belong on Page 2 (they support *doing* the
   exercise).
2. **Missing content**: Page 1 has no actual teaching content (a short
   explanation of the grammar point / communication skill the exercise is
   built around). `learningGoal`, `skillFocus`, `instructions`,
   `teachingNote`, `instructionInSourceLanguage` are the closest things to
   "teach" content today, but they're thin (1-2 sentences) and mixed in with
   exercise fields in the same `contentJson` blob / AI prompt output.

## Scope options

**Option A — Move-only (small, ships now):**
Move the exercise-setup fields (`scenario`, `situation`, `targetPhrases`,
`targetVocabulary`, `exampleText`, `commonMistakeToAvoid`, `audience`, `tone`,
`speakerRole`, `listenerRole`) from `teachContent()` to `practiceContent()`,
and render them in `ActivityPracticePageComponent` above
`app-exercise-renderer`. Page 1 keeps only `learningGoal`, `skillFocus`,
`instructions`, `teachingNote`, `instructionInSourceLanguage`,
`targetVocabulary`/`targetPhrases` *if reframed as "what you'll learn"*. No AI
prompt changes, no backend changes — pure frontend presenter + template
move. Page 1 becomes thinner but no longer shows exercise/task details.

**Option B — Move + new "Learn" content (matches user's example exactly):**
Option A, plus extend the unified `{teach, practice}` AI prompt contract
(already proposed in the Step 1-5 master plan, Phase 2) so `teach` includes a
real micro-lesson: a short grammar/communication explanation (e.g. "how to
form conditional sentences" / "how to politely ask for something") generated
from the pattern's `teachingPurpose` + the exercise's specific grammar/skill
focus. Requires AI prompt changes + calibration for all 10 patterns — a real
sprint, per the Phase 2 migration-order guidance (steps 3-5 of that plan took
multiple sessions each).

Option A directly fixes the user's stated complaint ("nothing about task or
exercise on first page") with a same-day frontend change. Option B is the
fuller vision ("teach conditional grammar / polite requests") but is a
multi-day AI content sprint on top of A.

## Recommendation

Ship Option A first (today), as Step 6 of the existing migration — it's a
pure presenter/template move, zero new AI prompts, fixes the literal "page 1
has exercise content" bug. Then scope Option B as its own sprint (AI prompt +
`{teach, practice}` contract work), since it requires prompt calibration per
the Phase 2 plan's stated risk.

## Decision (2026-06-14)

User chose: Option A now (move exercise-setup fields from Page 1 to Page 2),
plus create a separate sprint doc for Option B (AI teach-content generation
sprint) as a tracked follow-up.
