# Phase 11C: Mastery Routing and Review Scaffold Integration

**Date:** 2026-06-26
**Status:** Complete

## Goal

Wire the mastery engine into curriculum routing, readiness replenishment, and lesson/practice generation so that mastered objectives are excluded from new-learning routes and review-eligible objectives can feed review paths.

## What Was Implemented

### Part B — RoutingMode enum

New file: `src/LinguaCoach.Application/Curriculum/RoutingMode.cs`

- `NewLearning` — exclude mastered objectives (default)
- `Review` — include NeedsReview / reviewable mastered objectives
- `Reinforcement` — for AtRisk/NeedsPractice objectives
- `Preview` — admin preview, no state mutation

Added `Mode` property to `CurriculumRoutingRequest`.

### Part C — Factory and call site wiring

`CurriculumRoutingRequestFactory.Build()` now accepts:
- `masteredObjectiveKeys` — keys the routing service will exclude (or allow for review)
- `allowReviewOfMastered` — when true, reviewable mastered objectives are allowed
- `mode` — `RoutingMode` enum value

Call sites updated to fetch mastered keys and pass them:

| Call site | File | Change |
|-----------|------|--------|
| ReadinessPoolReplenishmentService | `ReadinessPool/ReadinessPoolReplenishmentService.cs` | Injects `IStudentMasteryEvaluationService`; fetches mastered keys before FillShortfallAsync loop; passes keys + mode |
| LessonBatchGenerationJob | `Jobs/LessonBatchGenerationJob.cs` | Injects mastery service; fetches keys in BuildCompactSummaryAsync; passes `RoutingMode.NewLearning` |
| PracticeGymGenerationJob | `Jobs/PracticeGymGenerationJob.cs` | Injects mastery service; fetches keys in MaterializeAsync; passes `RoutingMode.NewLearning` |

Synchronous page-load paths (ExercisePrepareHandler, ActivityGetHandler) intentionally not changed — fetching mastery state synchronously on every page load would add latency with no benefit; mastery is already applied via the background replenishment pool.

### Part D/E — Review scaffold and pool count separation

- `PoolHealthSummary.ShortfallCount` already excludes `ReviewOnly` from the new-learning target count. No change needed.
- When `EnableReviewScaffoldGeneration=true` and weak events exist, `allowReviewOrScaffold=true` now also sets `RoutingMode.Review` and `allowReviewOfMastered=true`.
- Review-route items come back with `IsLowerLevelContent=true` and appear in the PracticeGym review section via existing `PracticeGymSuggestionService` logic.

### Part I — Non-runnable skill guardrail in routing

`CurriculumRoutingService.RecommendAsync` now calls `FilterNonRunnable()` before `FilterByMastered()`.

- Grammar, Pronunciation, Fluency, Confidence objectives are filtered out of routing candidates.
- If all candidates are non-runnable, an empty list is returned and routing falls back to general_english.
- Warning logged for each non-runnable objective filtered.
- Existing validation warnings in `CurriculumValidationService` remain unchanged.

Integration test fix:

## Remaining Gaps

- `ExercisePrepareHandler` and `ActivityGetHandler` (synchronous page-load paths) do not fetch mastered keys. Acceptable for now — mastery-aware pool items are pre-generated by replenishment.
- `EnableReviewScaffoldGeneration` defaults to `false`. Review paths are implemented but gated. Enable after mastery signals are validated in production.
- Grammar/Pronunciation objectives remain in the syllabus seeder but will not be selected for generation (warning logged).
- Admin preview endpoint does not yet expose `RoutingMode` selector in the request DTO — minor gap, can be added when admin UI needs it.
